 PART 1 ─ WHY REDESIGN: THE V1 PROBLEMS
═══════════════════════════════════════════════════════════════════════════════════════════════════════════

  PROBLEM 1: Two declaration systems that don't talk to each other
  ┌─────────────────────────┐          ┌─────────────────────────┐
  │  DeclarationEditor.tsx  │          │  StallionWorkbench.tsx   │
  │  ─────────────────────  │          │  ──────────────────────  │
  │  zustand + localStorage │    ✗     │  Backend JSON file       │
  │  /declarations/validate │◄──NO───►│  /pack/generate          │
  │  /declarations/export   │ LINK    │  /declarations API       │
  └─────────────────────────┘          └─────────────────────────┘
         │                                     │
         ▼                                     ▼
  Home page reads HERE             BrokerReview reads HERE
  (localStorage)                   (backend /declarations)
                  ▲
                  │
          ╔═══════════════╗
          ║  RESULT:      ║
          ║  Declarations ║
          ║  created in   ║
          ║  one are      ║
          ║  invisible    ║
          ║  in the other ║
          ╚═══════════════╝

  PROBLEM 2: File-based JSON store with no locking
  ┌──────────────────────────────────────────────────┐
  │  data/declarations.json                          │
  │  ──────────────────────────────────────────────  │
  │  load_declarations() ──► full file read          │
  │  modify in memory                                │
  │  save_declarations() ──► full file write         │
  │                                                  │
  │  Two concurrent PATCHes = DATA LOSS              │
  │  Server restart = fine (file persists)            │
  │  Two users at once = race condition              │
  └──────────────────────────────────────────────────┘

  PROBLEM 3: Port number chaos
  ┌────────────────┐  ┌────────────────┐  ┌────────────────┐
  │  README: 8020  │  │  Scripts: 8021 │  │  PM2: 8022     │
  └────────────────┘  └────────────────┘  └────────────────┘

  PROBLEM 4: 30+ files generated, deployment status unknown
  ┌──────────────────────────────────────────────────────────────────────────┐
  │  BrokerReview v1 → v2 → v3 → v4 → production stub → production full   │
  │  DeclarationsList v1 → v2                                               │
  │  Workbench monolith → 6-component split                                │
  │  xml_builder.py → hotfix v1 → hotfix v2                                 │
  │  pack_service.py → bug fix v1 → LB01 rebuild v2                        │
  │  models.py → v2 → v3                                                    │
  │  worksheet_service.py → v2 → v3                                         │
  │                                                                          │
  │  WHICH VERSION IS LIVE? ──► Unknown for most files                      │
  └──────────────────────────────────────────────────────────────────────────┘

  PROBLEM 5: No auth, no multi-tenancy
  ┌──────────────────────────────────────────────────┐
  │  Anyone with the URL sees all declarations       │
  │  No user accounts                                │
  │  No client isolation                             │
  │  Blocks every client after UPSL                  │
  └──────────────────────────────────────────────────┘

  PROBLEM 6: Critical features built but never tested
  ┌──────────────────────────────────────────────────┐
  │  Document extraction ──► code exists, 0 tests    │
  │  Spreadsheet ingestion ──► discussed, 0 code     │
  │  CBTT rate lookup ──► button exists, hardcoded   │
  │  HS code lookup ──► discussed, 0 code            │
  │  Register CSV export ──► discussed, 0 code       │
  │  WhatsApp alerts ──► discussed, 0 code           │
  └──────────────────────────────────────────────────┘

 SUMMARY
═══════════════════════════════════════════════════════════════════════════════════════════════════════════

  The V1 Stallion codebase has a working core engine (ASYCUDA accepted
  the XML) and well-designed UI components. The problems are structural,
  not conceptual:

    1. Two data systems that don't talk to each other
    2. No auth or multi-tenancy
    3. Too many features started, not enough finished
    4. Deployment confirmation gaps across 30+ files
    5. Critical bugs discovered late (HS dots, FOB key, element order)
       because the reference XML was obtained too late

  If starting over, the single biggest change is:

    GET THE REFERENCE XML FIRST. THEN BUILD.

  Every hotfix session, every structural diff, every "ASYCUDA rejected"
  moment traces back to building the XML generator without having a real
  accepted XML to compare against. That one decision cost more time than
  any other technical choice in the project.

  The second biggest change is: ONE store, ONE model, gates between phases.
  Complete each layer before starting the next. Confirm deployment before
  moving on. The V1 breadth is impressive ─ the V2 depth would be sellable.
